# ラジオ投稿管理アプリ - フロントエンド

## 概要
日常のメモを蓄積し、LLM (Google Gemini) を利用して、登録されたラジオ番組の最適な「コーナー」へ振り分け、メール投稿を管理するWebアプリケーションのフロントエンド。

## 技術スタック
- **Frontend Framework:** Streamlit
- **Language:** Python
- **Data:** モックデータ（バックエンドAPI実装前）

## ディレクトリ構造
```
frontend/
├── app.py                          # メインアプリ（ダッシュボード）
├── pages/                          # ページディレクトリ
│   ├── 1_📝_メモ一覧.py           # メモ一覧・検索
│   ├── 2_📻_番組管理.py           # 番組・コーナー管理
│   ├── 3_✉️_メール作成.py         # メール下書き作成
│   └── 4_👤_プロフィール設定.py    # プロフィール管理
└── utils/                          # ユーティリティ
    ├── mock_data.py                # モックデータ
    └── styles.py                   # カスタムCSS
```

## 機能一覧

### 1. ダッシュボード (`app.py`)
- クイックメモ入力
- 投稿統計の表示（下書き/送信済み/採用/不採用）
- 最近のメモ一覧

### 2. メモ一覧 (`pages/1_📝_メモ一覧.py`)
- メモの一覧表示
- 検索・フィルタリング機能
- 新規メモ作成
- メモからメール作成への遷移

### 3. 番組管理 (`pages/2_📻_番組管理.py`)
- 番組一覧の表示
- パーソナリティでの絞り込み
- 番組名での部分一致検索
- 新規番組登録（番組情報・コーナー設定）
- 各番組のコーナー一覧表示

### 4. メール作成 (`pages/3_✉️_メール作成.py`)
- メモの選択
- AI推奨コーナーの表示（モック）
- 手動でのコーナー選択
- プロフィール選択
- メール件名・本文の編集
- ステータス管理
- メーラー起動（mailto連携）

### 5. プロフィール設定 (`pages/4_👤_プロフィール設定.py`)
- プロフィール一覧
- 新規プロフィール作成
- プロフィール情報管理（ラジオネーム、本名、住所、電話番号）
- メール署名プレビュー

## セットアップ

### 必要な環境
- Python 3.10以上
- Pipenv

### インストール
```bash
cd frontend
pipenv install
```

### 起動方法
```bash
cd frontend
pipenv run streamlit run app.py
```

アプリは `http://localhost:8501` で起動します。

## モックデータ

バックエンドAPI実装前のため、現在は `utils/mock_data.py` で定義されたモックデータを使用しています。

### モックデータの内容
- **ユーザー:** 1件
- **プロフィール:** 2件
- **パーソナリティ:** 4件
- **番組:** 3件
- **コーナー:** 7件
- **メモ:** 5件
- **メール:** 3件

## 今後の実装予定

### バックエンドAPI連携
現在はモックデータですが、以下のAPIエンドポイントを実装予定：

- `GET /api/memos` - メモ一覧取得
- `POST /api/memos` - メモ作成
- `GET /api/programs` - 番組一覧取得
- `POST /api/programs` - 番組作成
- `GET /api/corners` - コーナー一覧取得
- `POST /api/mails` - メール作成
- `GET /api/profiles` - プロフィール一覧取得
- `POST /api/analyze` - LLMによるメモ解析

### LLM機能
- Google Gemini APIを使用したメモの自動分析
- 最適なコーナーの推奨（一致度スコア付き）

## 開発ガイド

### 新しいページの追加
1. `pages/` ディレクトリに新しいPythonファイルを作成
2. ファイル名は `番号_絵文字_ページ名.py` の形式
3. 必要に応じて `utils/mock_data.py` にデータを追加

### スタイルのカスタマイズ
`utils/styles.py` の `get_custom_css()` 関数内でCSSを編集できます。

## 参考デザイン
`stitch_program_segment_management 2/` ディレクトリ内のHTMLファイルを参考にデザインを実装しています。
